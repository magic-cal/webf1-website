Style (work it!)

Working with stylesheets today to change the appearance of the site. By using sets of rules that override one another, you can specify styles for different elements.

By using style sheets (CSS) it makes it easier to update and change things in the site as all of the data controlling many elements is all in one place.

Although stylesheets are very useful, they should be linked into the website and should not be embedded. you can line a stylesheet like this:

Whats the address?

We focused on where each element points to within a site. Structure is very important so that each section points to the correct file.

Index.html is found first and is shown to the client. Other specific file names are picked up by the http protocol. Alternatve to http, here are some other protocols:

  • ftp://
  • http://
  • https://
  • file://
  • mailto: < this is used to send emails

The two types of links (relative and absolute) are important as they can reference files from wherever it is stored as apposed to just one place where it is saved. 

//This is what a relative link looks like:
      <a href="discovery-log.html">Discovery Log</a>
//This is what an absolute link looks like:
      <a href="http://825573.web1.rdfx.org/discovery-log.html">Discovery Log</a>

Although these are both linking to the same location, the top one is linking based on where it is already and the second will always point to the same place. On our VMs the second won't work as the links will be different depending on where it is hosted.
